# 収支ダッシュボード — Cursor 引き継ぎプロンプト

## プロジェクト概要

マネーフォワードMEからエクスポートしたCSVを読み込み、収支を可視化するWebツール。  
GitHub Pagesで公開することを前提とした **単一HTMLファイル構成**。

---

## これまでの経緯・決定事項

### ユーザーの目的
- 毎月の収支（収入・支出）を自動集計・可視化したい
- マネーフォワードME（家計簿アプリ）と連携したい
- GitHub Pagesにオープンに公開してツールとして使いたい

### 自動連携について（検討済み・断念）
- マネーフォワードMEは個人向けAPIを公開していない
- スクレイピングは利用規約違反リスクあり・セキュリティ上も不適切
- → **CSVダウンロード → ブラウザにドロップ** という運用で解決

### CSVの取得方法（確定）
マネーフォワードME → 収支グラフ → 明細 → CSVダウンロード

### 実際に使ったCSVの仕様（マネーフォワードME形式）
- 文字コード: Shift-JIS（cp932）
- 列: `計算対象`, `日付`, `内容`, `金額（円）`, `保有金融機関`, `大項目`, `中項目`, `メモ`, `振替`, `ID`
- 金額: 支出がマイナス値、収入がプラス値
- 振替フラグ=1 の行は除外すること
- 計算対象=0 の行は除外すること

## 現在の実装（`index.html` / `kakeibo.html`）

### 技術スタック
- **純粋なHTML/CSS/JS**（フレームワークなし）
- Chart.js 4.4.1（CDN）
- Google Fonts: Noto Sans JP + DM Mono

### 画面構成（3タブ）

#### 1. CSV取り込みタブ
- ドラッグ＆ドロップ or クリックでファイル選択
- **複数ファイル同時読み込み・追加読み込み対応**
- 読み込んだファイルをチップ表示 → 個別削除可能
- 月フィルター / 収入・支出フィルター
- 収入合計・支出合計・収支バランスのメトリクス表示
- 月別収支棒グラフ（Chart.js）
- 支出カテゴリ別ランキングバー（上位10件）
- 明細一覧テーブル（新しい順、最大500件）

#### 2. 手動入力タブ
- 収入・支出それぞれ項目名と金額を自由入力
- 行の追加・削除が可能
- リアルタイムで収支バランスを計算・表示

#### 3. ダッシュボードタブ
- CSVデータと手動入力を合算したサマリー
- 月別収支グラフ
- 支出カテゴリランキング

### デザイン仕様
- ダークテーマ（背景: `#0f0f0f`）
- アクセントカラー: `#e8d5b0`（ゴールド系）
- 収入: `#4ade80`（緑）、支出: `#f87171`（赤）
- スティッキーヘッダー（読み込み済みデータ期間・件数を表示）
- モバイル対応（720px以下でグリッド1列に変更）

### 主要な関数

| 関数 | 役割 |
|------|------|
| `parseCSV(text)` | CSV文字列を行オブジェクト配列に変換 |
| `detectFields(rows)` | ヘッダー名から列を自動判定 |
| `processCSV(raw, filename)` | 正規化されたトランザクション配列に変換 |
| `loadFiles(files)` | FileReader でファイル読み込み（UTF-8） |
| `applyCSVFilter()` | 月・タイプでフィルタリングして再描画 |
| `renderBarChart(...)` | Chart.js で月別棒グラフを描画 |
| `renderCatBars(id, rows)` | カテゴリ別支出バーを描画 |
| `renderTable(rows)` | 明細テーブルを描画 |
| `renderManual()` | 手動入力フォームを描画 |
| `renderDashboard()` | ダッシュボードタブを描画 |

### データ構造（トランザクション1件）
```js
{
  date:  "2025/01/15",   // 日付（文字列）
  name:  "スーパー",      // 内容
  cat:   "食費",          // 大項目カテゴリ
  amt:   -1200,           // 金額（収入=正、支出=負）
  type:  "exp",           // "inc" or "exp"
  bank:  "楽天カード",    // 保有金融機関
  month: "2025/01",       // YYYY/MM（フィルター用）
}
```

## 既知の課題・改善したいこと（Cursorで対応してほしい内容）

以下はユーザーから明示的に要望があった or 会話の中で出てきた改善候補:

1. **データの永続化**  
   現状はページリロードでデータが消える。`localStorage` や `IndexedDB` で保持したい。

2. **CSVのShift-JIS対応**  
   現在は `FileReader.readAsText(file, 'UTF-8')` で読んでいる。  
   マネーフォワードのCSVはShift-JIS（cp932）なので、文字化けが起きる可能性がある。  
   → `TextDecoder('shift_jis')` を使ったバイナリ読み込みに切り替えるべき。

3. **月別収支の折れ線グラフ追加**  
   棒グラフに加えて、収支推移の折れ線グラフも欲しい。

4. **カテゴリ別の月次推移**  
   「食費が月ごとにどう変化しているか」を見たい。

5. **レスポンシブ改善**  
   スマホでの明細テーブルが見づらい。

6. **エクスポート機能**  
   フィルタ後のデータをCSVでダウンロードしたい。

7. **収支サマリーのコメント自動生成**  
   AI（Claude API等）を使って「今月は食費が先月比+15%です」などのインサイトを自動生成したい（将来的な要望）。
